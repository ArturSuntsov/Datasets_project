## Как построился рейтинг Алексея, Марии и Дмитрия

### Исходные данные

У каждого аннотатора был **начальный рейтинг**, заданный при создании:

| Аннотатор | Начальный рейтинг |
|-----------|:-----------------:|
| Алексей (`annotator_alex`) | 4.0 |
| Мария (`annotator_maria`) | 4.1 |
| Дмитрий (`annotator_dmitry`) | 4.2 |

---

### Задача (CV Task)

- **Сложность (difficulty_score):** 0.7
- **Формат:** classification_v1
- **Метки:** `cat` / `dog`

**Ответы аннотаторов:**

| Аннотатор | Ответ |
|-----------|-------|
| Алексей | `cat` |
| Мария | `cat` |
| Дмитрий | `dog` |

---

### Шаг 1: Dawid-Skene (EM-алгоритм)

Алгоритм запустился с тремя аннотаторами и двумя метками.

**После 5 итераций сошёлся** (`converged: true`).

**Confusion matrix для каждого:**

| Аннотатор | Истина `cat` → `cat` | Истина `cat` → `dog` | Истина `dog` → `cat` | Истина `dog` → `dog` |
|-----------|:--------------------:|:--------------------:|:--------------------:|:--------------------:|
| Алексей | 100% | 0% | 100% | 0% |
| Мария | 100% | 0% | 100% | 0% |
| Дмитрий | 0% | 100% | 0% | 100% |

**Accuracy (доля правильных ответов):**

- Алексей: `accuracy = 0.5` (диагональ матрицы: правильно `cat`, неправильно `dog`)
- Мария: `accuracy = 0.5`
- Дмитрий: `accuracy = 0.5`

Почему у всех 0.5? Потому что 2 голоса за `cat`, 1 за `dog` — алгоритм не может определить, кто прав (нет явного большинства). Поэтому `final_label = ""` (пусто), и все получили нейтральную оценку.

---

### Шаг 2: Расчёт task_score

Формула:

```
complexity_weight = 0.5 + 0.5 × difficulty_score
                  = 0.5 + 0.5 × 0.7
                  = 0.85

task_score = accuracy × complexity_weight
           = 0.5 × 0.85
           = 0.425
```

У всех троих одинаковый `task_score = 0.425`.

---

### Шаг 3: Обновление рейтинга через EWMA

Формула:

```
rating_new = α × task_score + (1 − α) × rating_old
           = 0.1 × 0.425 + 0.9 × rating_old
           = 0.0425 + 0.9 × rating_old
```

Где `α = 0.1` (ANNOTATOR_RATING_ALPHA).

---

#### Алексей (начальный рейтинг 4.0)

**После 1-го прогона:**

```
rating_new = 0.0425 + 0.9 × 4.0
           = 0.0425 + 3.6
           = 3.6425
delta = 3.6425 − 4.0 = −0.3575
```

**После 2-го прогона (тот же task_score = 0.425):**

```
rating_new = 0.0425 + 0.9 × 3.6425
           = 0.0425 + 3.27825
           = 3.32075 → округлено до 3.3208
delta = 3.3208 − 3.6425 = −0.3217
```

**Текущий рейтинг: 3.03** (ещё несколько прогонов или другие задачи снизили дальше).

---

#### Мария (начальный рейтинг 4.1)

**После 1-го прогона:**

```
rating_new = 0.0425 + 0.9 × 4.1
           = 0.0425 + 3.69
           = 3.7325
delta = 3.7325 − 4.1 = −0.3675
```

**После 2-го прогона:**

```
rating_new = 0.0425 + 0.9 × 3.7325
           = 0.0425 + 3.35925
           = 3.40175 → округлено до 3.4017
delta = 3.4017 − 3.7325 = −0.3308
```

**Текущий рейтинг: 3.10**.

---

#### Дмитрий (начальный рейтинг 4.2)

**После 1-го прогона:**

```
rating_new = 0.0425 + 0.9 × 4.2
           = 0.0425 + 3.78
           = 3.8225
delta = 3.8225 − 4.2 = −0.3775
```

**После 2-го прогона:**

```
rating_new = 0.0425 + 0.9 × 3.8225
           = 0.0425 + 3.44025
           = 3.48275 → округлено до 3.4827
delta = 3.4827 − 3.8225 = −0.3397
```

**Текущий рейтинг: 3.18**.

### Почему рейтинг падает

`task_score = 0.425` ниже текущего рейтинга у всех (~4.0), поэтому EWMA тянет рейтинг вниз. Это правильно: аннотаторы пока не показали высокую accuracy, система понижает доверие к ним.

**Чтобы рейтинг рос:** нужно создать задачу, где ответы аннотаторов совпадут с консенсусом (accuracy = 1.0). Тогда `task_score = 1.0 × 0.85 = 0.85`, и рейтинг вырастет.
